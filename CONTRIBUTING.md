# Contributing

Thank you for your interest in contributing to Egov.Integrations.ClientAuthentication!

## Code of Conduct

By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md).

## How to Contribute

1.  **Report Bugs**: Use GitHub Issues to report bugs. Provide as much detail as possible, including steps to reproduce.
2.  **Suggest Features**: Use GitHub Issues to suggest new features or improvements.
3.  **Submit Pull Requests**:
    *   Fork the repository.
    *   Create a new branch for your changes.
    *   Ensure your code follows the existing style.
    *   Add tests for any new functionality.
    *   Ensure all tests pass.
    *   Submit a pull request with a clear description of the changes.

## Development Setup

1.  Clone the repository.
2.  Open the solution `src/Egov.Integrations.ClientAuthentication.sln` in your favorite IDE (Rider, Visual Studio, or VS Code).
3.  Build the project: `dotnet build src/`.
4.  Run tests: `dotnet test src/`.

## Commit Messages

Use clear and concise commit messages. We recommend following the [Conventional Commits](https://www.conventionalcommits.org/) specification.
